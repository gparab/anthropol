# Hardware-Biological Binding Protocol (Anthropol.io)

## Threat Model: Virtual Camera Injection
Virtual cameras (e.g., OBS Virtual Cam, ManyCam) can inject arbitrary pixel buffers into browser-based media streams. This bypasses traditional visual liveness checks because the "replay" happens at the OS-driver level, appearing as a valid hardware device to the browser.

## Defense Mechanism: Enclave-Attested Biometrics
To defeat this, we bind the **biological signals** (extracted via POS rPPG) to the **physical device fingerprint** via WebAuthn.

### Client-Side Process (HardwareBindingService)
1. **Biological Sampling**: The `BioOpticEngine` extracts the heart rate frequency vector ($S$) from the user's face.
2. **Hashing**: We generate a SHA-256 hash of the specimens: $H = \text{Hash}(S)$.
3. **Commitment**: We generate a unique transaction nonce ($N$).
4. **Signature**: We prompt the device's Secure Enclave (FaceID/TouchID) to sign a combined challenge: $\text{Sign}_{\text{TPM}}(\text{Hash}(N + H))$.
   - We enforce `authenticatorAttachment: "platform"` (not used in `navigator.credentials.get` but implied by binding to a platform-generated passkey).

### Server-Layer / Oracle verification
The Node.js Master Oracle performs the following check to guarantee data integrity:

1. **Reconstruct Challenge**: The Oracle receives $N$, $H$, and the WebAuthn signature.
2. **Verify Signature**: It validates that the signature was generated by the registered `credentialId` using the known public key.
3. **Integrity Check**: It confirms that the signed challenge matches $\text{Hash}(N + H)$.
4. **Biological Cross-Reference**: It verifies that $H$ actually corresponds to the biometric telemetry samples provided in the same payload.

**Result**: If an attacker attempts to substitute the real user's heart rate with a fake one (or a replay), they cannot generate a valid `Sign_TPM` commitment without physical access to the device's hardware enclave, effectively anchoring the biological data to a specific, trusted piece of hardware.
